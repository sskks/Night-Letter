## 3. 产品里程碑

### M0：可演示原型

状态：已达到。

验收标准：

- 可以打开页面并体验写梦、回信、历史、星图、基础广场。
- 视觉风格有辨识度，适合展示产品方向。

### M1：真实账号可用

状态：进行中，未完成。

验收标准：

- 用户 A 注册/登录后写梦，刷新页面和重新登录后数据仍在。
- 用户 B 注册/登录后看不到 A 的私有梦境。
- 同一个账号在重新登录后，梦境列表不重复、不丢失。
- 匿名体验不会混入真实账号数据。

### M2：真实共鸣角双用户闭环

状态：未完成，是下一轮 P0。

验收标准：

- 用户 A 分享一个梦境片段到共鸣角。
- 用户 B 登录后能在共鸣角看到 A 的片段。
- 用户 B 点星后，刷新/重登后仍保持“已点星”，再次点击可取消。
- 用户 B 发送“我也懂”后，评论立刻显示在卡片下方。
- 用户 A 重新登录后能收到 B 的回应提示。
- demo 内容只用于未登录预览，不参与真实互动。

### M3：可持续迭代产品

状态：待开始。

验收标准：

- 有明确验收清单。
- 每轮开发后更新 `DEVELOPMENT-TRACKER.md`。
- 关键路径用浏览器实际验证，不只看代码。
- 生产风险如 API Key、权限规则、数据删除都有处理策略。

## 4. 下一轮开发计划

### Sprint 3A：账号与云端数据稳态

优先级：P0  
目标：解决“每次重登账号就没有当前账号数据”的不确定性。

执行基线：

- 详细工程计划见 [SPRINT-3A-PLAN.md](/D:/vscode/Microsoft%20VS%20Code/project/Dream/SPRINT-3A-PLAN.md)
- 账号链路 UI 升级范围见 [UI-BRIEF-SPRINT-3A.md](/D:/vscode/Microsoft%20VS%20Code/project/Dream/UI-BRIEF-SPRINT-3A.md)

任务：

- 统一 CloudBase 用户标识，优先使用 `uid / uuid / openid`，不要只依赖邮箱或本地随机值。
- 本地缓存 key 按稳定用户 id 隔离。
- 登录、注册、页面刷新、切换账号都走同一套 `finishCloudLogin()` 流程。
- 重登后先拉云端数据，再与本地合并，最后刷新首页。
- 梳理匿名体验：匿名只能体验基础功能，真实共鸣角必须要求注册登录。

验收：

- 账号 A 写 2 条梦，退出再登录仍显示 2 条。
- 账号 B 登录后不显示账号 A 的私有梦。
- 账号 A 重复登录 3 次，CloudBase `dreams` 不产生重复 `localId`。

### Sprint 3B：共鸣角真实互动

优先级：P0  
目标：两个真实用户可以互相点星、回应、收到通知。

任务：

- `publicDreams` 增加字段：`ownerId`、`ownerName`、`stars`、`starBy`、`comments`。
- 点星逻辑改为：当前用户 id 在 `starBy` 中则取消，否则加入。
- 评论逻辑改为：写入当前用户昵称/匿名名，不再硬编码“匿名旅人”。
- 新增 `squareNotices` 集合：评论别人片段时给 owner 写通知。
- 首页/设置页读取未读通知，提示“有人回应了你的片段”。
- 未登录或匿名体验点击 demo 卡片时，明确提示“登录后才能真实互动”。

验收：

- 用户 B 点用户 A 的片段：星星 +1，刷新仍保留。
- 用户 B 再点一次：星星 -1，刷新仍保留。
- 用户 B 留言后，用户 A 登录能看到通知。
- 用户 A 点击通知后进入共鸣角，通知被标记已读。

### Sprint 3C：文案与界面一致性

优先级：P1  
目标：消除“广场/共鸣角”混用和登录注册旧文案。

任务：

- 全局把用户可见的“广场 / 梦境广场”统一为“共鸣 / 共鸣角”。
- 登录按钮文案统一为“登录夜信 / 创建账号”。
- 分享成功、分享失败、点星失败、回应失败文案统一成可理解的中文。
- 评论上限从 3 条调整到更合理的 10 条，或暂时不限制前 10 条展示。

验收：

- 底部 tab 显示“共鸣”。
- 页面标题显示“梦境共鸣角”或“共鸣角”。
- 登录注册没有旧式空格字样如“登 录 / 注 册”。

### Sprint 4：产品化基础设施

优先级：P1/P2  
目标：从“功能可用”走向“可以长期迭代”。

任务：

- Qwen API 迁移到 CloudBase 云函数，前端不再暴露密钥。
- 增加 PWA：`manifest.json`、Service Worker、添加到主屏。
- 增加基础内容安全：举报、隐藏、敏感词提示。
- 给共鸣角增加分页加载与情绪筛选。

## 5. CloudBase 数据模型草案

### `dreams`

用于用户私有梦境。

字段：

- `text`：梦境正文
- `emotion`：情绪
- `type`：梦境/白日梦/灵感
- `ts`：梦境时间
- `aiTurns`：AI 对话轮数
- `localId`：本地记录 id，用于去重
- `_openid`：CloudBase 自动注入

### `publicDreams`

用于共鸣角公开片段。

字段：

- `text`：公开片段
- `emotion`：情绪
- `type`：类型
- `ts`：原梦境时间
- `createTime`：分享时间
- `ownerId`：发布者稳定用户 id
- `ownerName`：发布者展示名
- `stars`：星星数量
- `starBy`：点星用户 id 数组
- `comments`：回应数组，包含 `userId`、`user`、`text`、`ts`
- `_openid`：CloudBase 自动注入

### `squareNotices`

用于共鸣角回应通知。

字段：

- `dreamId`：被回应的公开片段 id
- `ownerId`：接收通知的用户 id
- `ownerName`：接收通知的展示名
- `fromId`：回应者用户 id
- `fromName`：回应者展示名
- `text`：回应内容
- `ts`：回应时间
- `read`：是否已读

## 6. 每轮开发标准流程

每次开始开发前：

- 先读 `DEVELOPMENT-TRACKER.md`。
- 明确本轮只解决一个主目标。
- 写清楚验收标准。

每次改代码时：

- 优先改 `index.html` 这一唯一运行入口，不再维护根目录双 HTML 主入口。
- 不把 demo、本地兜底、真实云端互动混在一个判断里。
- 涉及 UI 的改动必须打开浏览器验证真实渲染状态。

每次结束前：

- 跑一遍本轮验收清单。
- 更新本文件的“当前事实快照”和 Sprint 状态。
- 记录未验证项，不把“代码写了”当作“产品完成”。

## 9. 本次检查结论

夜信已经是一个视觉和体验方向成立的 MVP，但当前不能把“广场/共鸣角已完成”视为产品完成。真正的下一关是账号和云端数据闭环：同一个账号重登不丢数据，不同账号能互相看到公开片段并产生可追踪的回应。

后续开发要以 M1 和 M2 为主线，先把真实用户链路跑通，再继续做传播、PWA、趋势图等增长和留存功能。
