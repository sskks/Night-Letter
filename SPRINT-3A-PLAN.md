## 1. Sprint 目标

Sprint 3A 只解决一件事：

**让同一个真实账号在登录、刷新、退出后重登、切换账号后，仍能稳定看到自己的云端数据，且不同账号之间数据不串。**

这一步完成前，不把“共鸣角真实互动”视为完成，不继续扩张分享、通知、趋势图等功能。

## 2. 当前代码真实状态

基于 [index.html](/D:/vscode/Microsoft%20VS%20Code/project/Dream/index.html) 当前实现，已经具备以下基础：

- 已接入 CloudBase 认证与数据库。
- 已有本地存储按账号隔离的雏形：`_getStoreKey()`、`migrateLegacyDreamsToCurrentUser()`。

## 4. 目标实现结构

### 4.1 稳定用户身份

新增统一规则：

1. 优先使用 CloudBase 会话中的稳定 id：`uid`
2. 其次 `uuid`
3. 其次 `userInfo.uid`
4. 都没有时，匿名态使用明确前缀 id，例如 `anon:<uid>`
5. 严禁只用邮箱当主键

建议在代码中收敛为一个工具函数，例如：

`getStableUserIdentity(sessionLike) -> { stableUserId, email, isAnonymous, mode }`

### 4.2 单一路径完成账号恢复

把以下入口统一收敛到一条完成态流程：

- 页面初始化时 SDK session 恢复
- 邮箱登录成功
- 注册成功后的补登录
- 匿名登录成功
- 切换账号后的重新登录

建议形成单一收口函数，例如：

`completeAuthSession(userIdentity, source)`

它负责按顺序执行：

1. 写入内存态用户缓存
2. 写入 `nightletter_session`
3. 计算当前账号的本地存储 key
4. 拉取云端 dreams
5. 与当前账号本地 dreams 去重合并
6. 回写本地
7. 刷新首页、历史、设置等渲染
8. 再关闭登录遮罩

### 4.3 本地数据隔离规则

本地存储至少分三层：

- 真实账号：`nightletter_dreams_user_<stableUserId>`
- 匿名云账号：`nightletter_dreams_anon_<stableUserId>`
- 本地试用：保留单独 key，不允许自动并入真实账号

原则：

- 真实账号不会读到本地试用数据，除非显式迁移。
- 切换账号时不复用上一个账号的 key。
- 登录后优先读当前账号 key，不再回退到共享 key。

### 4.4 云端合并顺序

统一为：

1. 读取云端 dreams
2. 读取当前账号本地 dreams
3. 以 `localId` 为第一去重键
4. 必要时增加 `text + ts` 兜底去重
5. 生成合并结果
6. 本地落盘
7. 仅在必要时补写云端缺失记录

禁止流程：

- 先用本地空数组覆盖云端
- 切账号时沿用旧账号缓存
- 登录成功后直接关遮罩但未完成数据恢复

## 5. UI 与文案范围

Sprint 3A 不做大改版，只做账号链路必要升级。

### 登录注册页面

- 强化“登录夜信 / 创建账号”的主次关系。
- 错误提示改为可执行中文，不直接暴露 SDK 原始英文。
- 明确匿名体验的边界：可浏览、可记录基础内容，但不保证真实云同步互动。

### 设置页账号区

- 已登录：显示当前账号、数据归属、切换账号、退出登录。
- 匿名体验：显示“当前是试用模式，注册后才会跨设备保存”。
- 未登录：显示“登录后可保存并恢复你的梦境记录”。

### 共鸣角入口提示

- 本 Sprint 只改提示，不改互动模型。
- 当用户未登录或处于本地试用模式时，分享入口需要明确提示：需要真实账号。

## 6. 任务拆分

### T1. 统一身份提取

- 收敛所有 `uid / uuid / userInfo.uid / email` 判断。
- 明确匿名态和本地试用态。

交付标准：

- 登录、注册、恢复流程都走同一身份结构。

### T2. 统一账号完成态流程

- 抽出单一 session 完成函数。
- 统一登录、注册后补登录、匿名登录、恢复登录的后置逻辑。

交付标准：

- 不再出现“注册成功但 UI 没切换”或“登录后没恢复数据”的分叉问题。

### T3. 本地缓存按账号彻底隔离

- 清理旧共享 key 的回退策略。
- 仅保留一次性迁移逻辑，不做长期混读。

交付标准：

- A/B 两个账号在同一浏览器中切换时，互不串数据。

### T4. 云端合并与补写顺序修正

- 登录恢复时先拉云端，再合并本地。
- 合并结果稳定后再决定是否补写云端。

交付标准：

- 刷新与重登不丢梦，不重复梦。

### T5. 账号链路文案优化

- 登录、注册、匿名体验、切换账号、错误提示统一。

交付标准：

- 所有账号链路文案都能直接解释“发生了什么”和“下一步怎么做”。

## 7. 验收用例

### A. 同账号重登

1. 用户 A 注册
2. 写入 2 条梦境
3. 刷新页面
4. 退出登录
5. 再次登录

通过标准：

- 仍显示同 2 条梦境
- 不新增重复记录

### B. 双账号隔离

1. 用户 A 登录并写梦
2. 退出
3. 用户 B 登录

通过标准：

- B 看不到 A 的私有梦境
- B 的历史列表为空或仅显示 B 自己的数据

### C. 连续恢复稳定性

1. 同一账号连续刷新 3 次
2. 连续退出再登录 3 次

通过标准：

- 本地列表数量稳定
- 云端 `dreams` 不产生重复 `localId`

### D. 匿名边界

1. 本地试用写 1 条梦
2. 不做显式迁移，直接登录真实账号

通过标准：

- 不自动污染真实账号数据
- 如有迁移行为，必须有明确提示
