# PRD — 夜信 Night Letter

> 产品需求文档 | 版本 1.0 | 2026-06-22

---

## 1. 需求陈述

「夜信」解决的核心问题是：**年轻人有记录梦境和情绪的需求，但缺少低门槛、有温度的工具。**

现有日记/冥想类产品往往过于功能化或过于严肃，而「夜信」通过"写信给梦"的隐喻，降低记录的心理门槛，并借助 AI 对话提供轻量的情绪觉察和陪伴。

**目标用户**：18-35 岁，关注自我探索、情绪管理、轻度心理疗愈的年轻人。

## 3. 用户流程

### 核心路径 A：首次使用
```
打开应用 → 引导页(Onboarding) → 注册/匿名登录 → 首页 → 写第一封信 → 收到AI回信
```

### 核心路径 B：日常使用
```
打开应用 → 签到心情 → 写信/对话 → 查看历史/星图
```

### 核心路径 C：社区互动
```
首页 → 广场 → 浏览他人梦境 → 点赞/评论 → 分享自己的梦
```

## 4. 页面结构

| 页面 | 路由 ID | 主要组件 |
|------|---------|---------|
| 首页 | `s-home` | 极光 Canvas、签到栏、连续天数、写信入口 |
| 写信 | `s-write` | 文字输入框、情绪标签、字数统计、语音输入、引导日记触发器 |
| 对话 | `s-chat` | 人格选择器、聊天气泡、输入框 |
| 收信 | `s-recv` | AI 解读卡片、情绪雷达图、导出按钮 |
| 星图 | `s-ns` | Canvas 可视化、梦境节点、分类筛选 |
| 历史 | `s-hist` | 搜索栏、情绪筛选标签、梦境卡片列表 |
| 广场 | `s-square` | 瀑布流卡片、点赞、评论、发布按钮 |
| 设置 | `s-set` | 账号信息、对话轮次、切换账号、数据管理 |

## 5. 验收标准

### F1 首页
- [ ] 页面加载后极光动画正常渲染（3 色块 + 12 粒子）
- [ ] 5 个心情 emoji 可点击，点击后高亮并保存到本地
- [ ] 连续天数正确计算（合并梦境记录 + 签到日期）
- [ ] 底部导航栏切换页面流畅

### F2 写信
- [ ] 文字输入框支持最多 500 字
- [ ] 6 个情绪标签可单选，选中后高亮
- [ ] 字数统计实时更新，超限时变红
- [ ] 点击"寄出"后数据正确保存

### F3 引导式日记
- [ ] 点击"不知道写什么？"触发引导流程
- [ ] 5 步卡片依次展示，支持跳过和返回
- [ ] 关闭按钮（X）可直接退出引导
- [ ] 完成后答案自动填入写信输入框

### F4 收信
- [ ] AI 回应卡片正确显示梦境解读
- [ ] 情绪雷达图根据检测结果渲染
- [ ] 导出按钮可生成图片卡片

### F5 对话
- [ ] 3 个人格（温柔倾听/理性观察/诗意旅人）可切换
- [ ] 人格选择持久化（刷新后保留）
- [ ] 达到轮次上限时温和结束（非硬切断）
- [ ] API 不可用时使用人格化本地回复

### F6 星图
- [ ] Canvas 正确渲染梦境节点
- [ ] 按情绪分类显示不同颜色
- [ ] 点击节点可查看梦境详情

### F7 历史
- [ ] 搜索框支持关键词搜索（200ms 防抖）
- [ ] 6 个情绪筛选标签可切换
- [ ] 搜索结果关键词高亮显示
- [ ] 副标题显示筛选数量（如 "3 / 10 封信"）
- [ ] 无结果时显示智能空状态提示

### F8 广场
- [ ] 瀑布流正确加载梦境卡片
- [ ] 点赞/评论功能正常
- [ ] 发布新梦境后列表更新
- [ ] 未登录时提示登录

### F9 设置
- [ ] 对话轮次设置（5/8/12/不限）即时生效
- [ ] 切换账号流程完整（退出 → 重新登录）
- [ ] 清除数据有二次确认

### F10 账号系统
- [ ] 邮箱注册/登录流程完整
- [ ] 匿名登录可正常使用基础功能
- [ ] 登录后自动同步本地数据到云端
- [ ] 不同账号的数据相互隔离

### F11 数据同步
- [ ] 登录后本地数据自动备份到 CloudBase
- [ ] 切换设备后数据可从云端恢复
- [ ] 离线时本地数据不受影响
