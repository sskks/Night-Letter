# 夜信 Night Letter - 梦境情绪后护理产品

## 产品概述

**夜信**是一款帮助用户通过记录梦境来觉察情绪、促进自我反思的情感后护理应用。

### 核心理念
> **梦不是答案，是情绪入口**

我们不给出标准答案，只做情绪镜像，尊重用户的自我思考能力。

---

## 目标用户
- 年轻成年人（18-35岁）
- 对梦境、情绪觉察、自我探索感兴趣
- 希望通过记录和理解梦境来获得情感慰藉

---

## 核心功能

### 1. 首页 - 沉浸星空
- 全屏 Canvas 渲染的沉浸式夜空
- 多层星星（200颗，分3层深度）
- 4团彩色星云缓慢漂移
- 柔光月亮（8倍外发光 + 12条射线）
- 金色漂浮粒子向上飘移
- 流星带彩色头部和长拖尾
- 触摸/鼠标视差交互

### 2. 写信页
- 简洁的文本输入界面
- "亲爱的夜："称呼营造亲密感
- 引导文案降低写作门槛
- 情绪标签输入（不安/想念/困惑/平静等）

### 3. 寄送动画
- 4圈扩散环动画
- 中心光点辐射效果
- "你的信正在路上，有人正在认真读它"文案

### 4. 回信页（情绪镜像）
- **单一回信内容**（无版本选择器）
- 结构：
  1. 映射用户状态（"你写下来的这些，不是为了让谁给你答案"）
  2. 描述动作本质（"把心里某个地方轻轻摊开"）
  3. 澄清梦的作用（"不会告诉你应该怎么做，但会让你感觉到白天被忽略的东西"）
  4. 肯定情绪价值（"这已经够了"）
  5. 底部轻轻提醒（温和的自我觉察引导）

### 5. 星图页
- Canvas 可视化12个梦境节点
- 情绪颜色编码（平静/不安/喜悦/思念/困惑）
- 星座连线脉冲动画
- **点击交互**：点击星星查看梦境详情弹窗
- 本周发现洞察卡片

### 6. 发现页
- 一句洞察（模式总结）
- 反复出现的意象统计
- 情绪天气分布
- 历史信件列表

## 技术实现

### 前端架构
- **单文件 HTML 原型**
- 纯 CSS + JavaScript（无框架依赖）
- Canvas 2D API 用于星空渲染
- 响应式设计（移动端优先 375-414px）

### 性能优化
- 星星数量从 300 降至 200
- 星云从 6 团减至 4 团
- 漂浮粒子从 35 个减至 20 个
- 流星生成频率降低（3秒/次）

### 视觉风格
- **深色梦幻美学**
- 主色调：深蓝/紫色调 (#050510, #0a0a1a)
- 强调色：暖金色 (#caa97a)
- 情绪色：
  - 平静: #6db8c4
  - 不安: #c47080
  - 喜悦: #c4b06d
  - 思念: #9472c4
  - 困惑: #7090c4

## 设计理念

### 为什么不做版本选择器？
用户反馈指出：不会有用户去选择不同标签的回答方式。产品核心是**情绪镜像 + 自我启发**，应该做到醒来就想记录，然后稍微给出一些启示提醒，重要的是尊重人的自我思考能力，自己启发自己。

### 为什么洞察文案要直白？
最初的"不安的海域"、"思念的丘陵"过于抽象，改为"你最近的梦反复出现两种情绪：不安和想念"更易懂。

### 为什么增加社区共鸣功能？
为了增加用户粘性，让用户看到其他人的梦如何与自己产生共鸣，形成情感连接。
