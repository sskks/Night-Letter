# PROJECT-BOUNDARIES.md — 夜信项目边界

> 版本 1.0 | 2026-06-22

---

## 1. 目标用户与场景

**用户画像**：18-35 岁，有自我探索意愿的年轻人。轻度使用，每次 3-10 分钟。

**使用场景**：
- 睡前/醒来后的梦境记录
- 情绪波动时的自我对话
- 碎片时间的轻度心理觉察
- 匿名分享梦境、获得社区温暖

**不是面向**：专业心理咨询师、重度失眠患者、需要医疗级干预的用户。

---

## 2. 性能要求

| 指标 | 要求 |
|------|------|
| 首屏加载 | < 3s（单文件 ~200KB + 字体 CDN） |
| 页面切换 | < 0.5s（CSS transition） |
| AI 响应 | < 8s（超时降级到本地回复） |
| 动画帧率 | 极光动画保持 30fps+ |
| localStorage | 单用户 < 5MB 限制内 |
| 云端同步 | 最多同步 500 条梦境 |

---

## 3. 安全与合规

- **不做**：不收集用户真实姓名、手机号等敏感信息
- **不做**：不提供心理诊断或治疗建议（仅为情绪觉察工具）
- **存储**：数据存在用户本地 + 腾讯云 CloudBase（中国大陆服务器）
- **API**：千问 API 密钥硬编码在前端（演示项目，非生产级）
- **匿名**：广场发帖完全匿名，无法追溯用户身份

---

## 4. 成本与资源限制

| 资源 | 限制 |
|------|------|
| CloudBase | 免费额度（基础版） |
| 千问 API | qwen-turbo 模型（低成本） |
| GitHub Pages | 免费静态托管 |
| 字体 CDN | fonts.loli.net（免费镜像） |
| 域名 | 无独立域名（GitHub Pages 子域名） |

---

## 5. 明确的"不做什么"

- ❌ **不做** 原生 App（iOS/Android）
- ❌ **不做** 后端服务器（纯前端 + Serverless）
- ❌ **不做** 实时通讯（对话是异步 AI 生成，非真人聊天）
- ❌ **不做** 付费/订阅功能
- ❌ **不做** 多语言国际化（仅中文）
- ❌ **不做** 暗黑/明亮主题切换（固定深色系）
- ❌ **不做** 框架迁移（保持纯 HTML/CSS/JS 单文件架构）
- ❌ **不做** 大规模并发支持（演示项目，非生产级）

---

## 6. 技术边界

| 约束 | 原因 |
|------|------|
| 单文件架构 | 项目定位是原型/MVP，降低复杂度 |
| 无构建工具 | 不引入 Webpack/Vite 等，保持零配置 |
| 无测试框架 | 手动验证为主（PRD 验收标准） |
| ES5/ES6 混合 | 兼容主流浏览器，不用最新语法 |
| CloudBase 唯一后端 | 不混用多个后端服务 |

---

## 7. 质量底线

- 所有核心功能（写信、收信、对话）必须有离线降级方案
- localStorage 操作必须有 JSON 解析错误保护
- AI 调用必须有超时（8s）和回调守卫（防重复触发）
- 数据删除操作必须有二次确认
- Git merge 禁止使用 `-X ours`（已知编码损坏风险）
